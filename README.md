# Архиватор на базе алгоритма Хаффмана

Консольный архиватор, реализующий сжатие и распаковку произвольных бинарных файлов с использованием статического алгоритма Хаффмана. Написан на C++17, собирается через CMake.

## Как это работает

Алгоритм Хаффмана строит оптимальный префиксный код: символы с большей частотой получают более короткие коды. Это позволяет эффективно сжимать данные с неравномерным распределением байт — тексты, исходный код, логи.

Формат архива `.huff`:
- 8 байт — общее число символов в исходном файле
- 2 байта — число уникальных символов
- таблица частот (1 байт символа + 8 байт частоты для каждого)
- закодированный битовый поток

Такой формат позволяет восстановить дерево при распаковке без хранения самих кодов.

## Сборка

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
```

Исполняемый файл появится в `build/src/huffman_archiver`.

## Использование

**Сжатие:**
```bash
./build/src/huffman_archiver -c <исходный_файл> <архив.huff>
```

**Распаковка:**
```bash
./build/src/huffman_archiver -d <архив.huff> <выходной_файл>
```

**Бенчмарк** (сжатие + распаковка + статистика):
```bash
./build/src/huffman_archiver -b <файл>
```

### Примеры

```bash
# Сжать текстовый файл
./build/src/huffman_archiver -c document.txt document.huff

# Распаковать
./build/src/huffman_archiver -d document.huff document_restored.txt

# Проверить что файлы совпадают
diff document.txt document_restored.txt

# Посмотреть степень сжатия
./build/src/huffman_archiver -b document.txt
```

## Тесты

```bash
cmake -B build -DBUILD_TESTS=ON
cmake --build build --parallel
ctest --test-dir build --output-on-failure
```

Тесты покрывают:
- побитовый ввод-вывод (BitWriter / BitReader)
- построение дерева: пустой вход, единственный символ, prefix-free свойство
- кодирование и декодирование через дерево
- полный цикл compress + decompress для пустого файла, одного символа, текста, всех 256 байт
- корректную обработку ошибок при отсутствующих файлах

## Эффективность сжатия

| Тип данных | Результат |
|---|---|
| Текст, исходный код | сжатие 40–55% |
| Уже сжатые данные (zip, jpg) | архив больше оригинала — ожидаемо |
| Случайные данные | архив больше оригинала — ожидаемо |

На случайных и уже сжатых данных энтропия близка к 8 битам на символ, поэтому статический алгоритм Хаффмана не даёт выигрыша — накладные расходы на заголовок увеличивают размер.

## Требования

- CMake 3.16+
- компилятор с поддержкой C++17 (GCC 9+, Clang 9+)
- интернет при первой сборке с тестами (скачивается Catch2)
